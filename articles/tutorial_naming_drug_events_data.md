# Adding Names to Drug Events Data

## Intro

This tutorial demonstrates how to work with drug events data using the
`FinnGenUtilsR` package. The drug events data contains medication
purchase and prescription records with VNR (Finnish national product
codes) identifiers. This tutorial shows how to append human-readable
medication names and ATC codes by joining with the `finngen_vnrs`
reference table.

Unlike other data types (such as service sector or longitudinal data),
there is no custom helper function for adding names to drug events data.
Instead, we use a simple join operation with the `finngen_vnrs` table.

To understand what the `fgbq` object is, see the dedicated vignette
`tutorial_fgbq`.

### Adding medication information via VNR codes

To add medication names and ATC codes to the drug events data, we simply
join with the `finngen_vnrs` reference table. The drug events table
contains two VNR columns:

- `MEDICATION_VNR`: The VNR code for the actual medication dispensed
- `PRESCRIPTION_VNR`: The VNR code from the original prescription (may
  differ from what was dispensed)

We can join on either column depending on which medication information
we need. Here we’ll join on `MEDICATION_VNR`:

Note: The VNR codes in the drug_events table are stored as strings,
while in the finngen_vnrs table they are integers. We need to convert
the type before joining.

``` r
drug_events_with_names <- drug_events_subject_tbl |> 
  mutate(MEDICATION_VNR = as.integer(MEDICATION_VNR)) |> 
  left_join(
    fgbq$tbl$finngen_vnrs,
    by = c("MEDICATION_VNR" = "VNR")
  )
```

with the service sector data, but the same principles apply to the drug
events data.
